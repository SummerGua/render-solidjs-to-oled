🔧 SolidJS → SSD1306 OLED 自定义渲染器

从 C 语言驱动到 JSX 响应式 UI — 技术全链路讲解

一、这个项目做了什么

用 SolidJS 写 JSX 组件，实时渲染到一块 128×64 单色 OLED 屏幕（SSD1306 芯片，I2C 通信），跑在树莓派上。

<!-- 和写 React 一模一样！ -->
<Screen>
  <Text x={0} y={0}>Hello OLED!</Text>
  <Text x={0} y={2}>Count: {count()}</Text>
  <Pixel x={42} y={32} />
</Screen>

信号变化 → OLED 自动刷新。和你在浏览器里写 Vue/React 体验一致。

🤔 核心问题：React/Vue/SolidJS 默认渲染目标是 浏览器 DOM。
怎么让它们渲染到一块 128×64 的 I2C 屏幕？

三、从底层讲起：C 驱动怎么工作

3.1 大一 C 语言就能看懂

OLED 屏幕通过 I2C（一种双线串行通信协议）连接到树莓派。SSD1306 芯片内部维护 1024 字节显存（128列 × 64行 ÷ 8位每字节 = 每字节管 8 个竖排像素）。

// SSD1306 显存结构（1024 字节 = 128 列 × 8 页）
// 每个字节的 bit0 = 该列最上面像素, bit7 = 最下面
//
// Page 0 (y=0~7):  [byte0  byte1  byte2  ... byte127]  ← 屏幕顶部 8 行
// Page 1 (y=8~15): [byte128 byte129 byte130 ... byte255]
// ...
// Page 7 (y=56~63):[byte896 ...              byte1023]  ← 屏幕底部 8 行

关键 C 函数（我们只用了 5 个）

函数作用类比前端
oledInit(channel, addr, type, flip, invert) 打开 I2C 设备文件，发送初始化指令序列给 SSD1306 new OLED()
oledFill(0x00) 全屏填充 0（清屏/黑屏） ctx.clearRect(0,0,128,64)
oledWriteString(x, y, "Hello", font) 在 (x,y) 写字，font=0/1/2 对应 8×8 / 16×24 / 6×8 字体 ctx.fillText("Hello", x, y)
oledSetPixel(x, y, 1) 点亮/熄灭单个像素 ctx.fillRect(x,y,1,1)
oledShutdown() 关显示，关 I2C cleanup()
🎯 关键发现：这些 C 函数是即写即刷新的。每个 oledWriteString 调用内部会直接 write(file_i2c, ...) 写到硬件。不需要单独 flush() 步骤。你可以理解为：每次 draw 调用都自动 commit。

四、JS 怎么调用 C 函数？FFI 桥接

Node.js 是 JavaScript 运行时，C 是编译型语言。它们之间需要一座桥：FFI（Foreign Function Interface，外部函数接口）。

我们用了 koffi 这个 npm 包（纯 JS 实现，不需要编译原生模块）。先把 C 代码编译成共享库：

# 把 .c 源文件编译成 .so 动态库
gcc -shared -fPIC -o liboled96.so oled96.c fonts.c -lm

# 含义：
# -shared   → 生成共享库（.so），类似 Windows 的 .dll
# -fPIC     → 位置无关代码（共享库必需的编译选项）
# -o xxx.so → 输出文件名
# -lm       → 链接数学库（libm）

然后 JS 这边用 koffi 加载并声明函数签名：

const koffi = require('koffi');
const lib = koffi.load('./oled_96/liboled96.so');

// 声明 C 函数：函数名、返回值类型、参数类型
const oledInit        = lib.func('oledInit', 'int', ['int','int','int','int','int']);
const oledFill        = lib.func('oledFill', 'int', ['uint8']);
const oledWriteString = lib.func('oledWriteString', 'int', ['int','int','string','int']);
const oledSetPixel    = lib.func('oledSetPixel', 'int', ['int','int','uint8']);
const oledShutdown    = lib.func('oledShutdown', 'void', []);

// 调用就像普通 JS 函数！
oledInit(1, 0x3C, 128, 0, 0);   // I2C总线1, 地址0x3C, 128x64, 不翻转, 不反色
oledFill(0);                       // 清屏
oledWriteString(0, 0, 'Hello!', 2); // x=0, y=0, 文字, FONT_SMALL
oledSetPixel(42, 42, 1);          // 点亮(42,42)
oledShutdown();                    // 关闭
⚠️ 踩坑记录：最早尝试用 ffi-napi，但在 Node 20 + ARM64 上编译失败（C++ 代码不兼容新版 node-api）。换成 koffi（纯 JS FFI，无需编译）解决了。

另一个坑：require('solid-js') 在 Node.js 默认加载 server.cjs（SSR 精简版，响应式系统是假的——createEffect 是空函数！）。必须显式引用 solid-js/dist/solid.cjs 才有完整响应式。

五、核心：自定义 JSX 渲染器

5.1 思路：和 React Reconciler 一个道理

React 用 react-reconciler 支持自定义渲染目标（如 React Native → 原生组件）。
我们的做法更简单：自定义 JSX pragma，把 JSX 编译结果拦截下来。

数据流：
JSX 源码
<Text x={0}>Hi</Text> ↓ babel + @babel/plugin-transform-react-jsx JS 函数调用
oled.jsx(Text, {x:0}, "Hi") ↓ 我们写的 jsx() 函数 元素描述对象
{type:'text', props:{x:0}, children:["Hi"]} ↓ renderElement() C 函数调用
oledWriteString(0, 0, "Hi", 2) ↓ I2C 总线 🖥️ OLED 屏幕显示 "Hi"

5.2 第一步：Babel 编译 JSX

在 .babelrc 中配置：

{
  "plugins": [
    ["@babel/plugin-transform-react-jsx", {
      "runtime": "classic",
      "pragma": "oled.jsx",          // <div> → oled.jsx("div", props)
      "pragmaFrag": "oled.Fragment"  // <> → oled.Fragment
    }]
  ]
}

JSX 编译结果（以 <Text x={0} y={2}>Count: {count()}</Text> 为例）：

// 编译前（你写的代码）
<Text x={0} y={2} font={SMALL}>Count: {count()}</Text>

// 编译后（babel 生成的 JS）
oled.jsx(Text, { x: 0, y: 2, font: SMALL }, "Count: ", count())

5.3 第二步：jsx() — 把调用变成元素树

function jsx(type, props, ...children) {
  const allProps = props || {};
  let kids = children.length > 0 ? children : allProps.children;

  // 大写开头 → 组件函数（如 Text, Pixel, Screen）
  if (typeof type === 'function') {
    return type({ ...allProps, children: kids });
  }

  // 小写 → 内置元素（如 'screen', 'text', 'pixel'）
  return { type, props: allProps, children: normalizeChildren(kids) };
}
React 开发者对比：
React: React.createElement('div', props, children) → 虚拟 DOM
我们: oled.jsx('text', props, children) → OLED 元素树
思路一模一样，换了渲染目标而已。

5.4 第三步：renderElement() — 遍历树画到 OLED

function renderElement(el) {
  if (typeof el === 'string' || typeof el === 'number') return;

  switch (el.type) {
    case 'screen':
      // 根容器，递归渲染子节点
      for (const child of el.children) renderElement(child);
      break;

    case 'text': {
      // 拼接所有子节点的文本，一次性写出
      let str = '';
      for (const child of el.children) {
        if (typeof child === 'string') str += child;
        else if (typeof child === 'function') str += String(child());
      }
      oledWriteString(el.props.x, el.props.y, str, el.props.font ?? 2);
      break;
    }

    case 'pixel':
      oledSetPixel(el.props.x, el.props.y, el.props.on !== false ? 1 : 0);
      break;
  }
}

六、响应式：signal 变了 OLED 自动刷新

这是最关键的部分。SolidJS 的 createSignal 提供响应式状态，但我们需要把"状态变化"和"OLED 刷新"连接起来。

6.1 关键原则：Signal 创建和 Signal 读取要分离

// ✅ 正确做法：
function CounterApp() {
  // 【创建】signal + 定时器 — 只执行一次
  const [count, setCount] = createSignal(0);
  setInterval(() => setCount(c => c + 1), 1000);

  // 【读取】返回一个函数，每次 effect 运行时调用
  return () => (
    <Screen>
      <Text x={0} y={2}>Count: {count()}</Text>
    </Screen>
  );
}
❌ 错误做法（踩过的坑）：
把 JSX 直接放在 return 语句中，导致 counterApp() 在 createRenderEffect 里重复调用。
每次重渲染都创建新的 createSignal + setInterval，信号泄漏，屏幕在正确/错误间疯狂跳变，最终乱码。
本质原因：就像 React 里不能在 render 中 useState，SolidJS 的 createSignal 也只能在组件初始化时调用。

6.2 render() 函数 — 把一切串起来

function render(code) {
  oledInit(1, 0x3C, 128, 0, 0);  // 初始化 OLED（一次）

  // ① 先执行 App() — 创建 signal + 定时器 — 只跑一次
  const getTree = code();  // 返回 () => JSX

  createRoot((dispose) => {
    // ② createRenderEffect 追踪 signal 变化
    createRenderEffect(() => {
      oledFill(0);                  // 清屏
      const tree = getTree();       // 调用 () => JSX，读取 signal 当前值
      renderElement(tree);          // 递归渲染到 OLED
    });
    // ③ signal 变了 → effect 自动重跑 → 屏幕更新 ✨
  });
}
完整响应式链路：
setInterval → setCount(c+1) ↓ SolidJS 内部：signal 值更新 + 通知 effect ↓ createRenderEffect 重跑：getTree() → 读 count() → 最新 JSX ↓ renderElement(tree) → oledFill(0) → oledWriteString(...) → oledSetPixel(...) ↓ I2C → OLED 屏幕更新 🎉

七、和 Vue/React 的对比

概念React / Vue (DOM)本项目 (OLED)
渲染目标 浏览器 DOM 树 SSD1306 显存 (1024字节)
创建元素 document.createElement('div') oled.jsx('text', props) → 元素对象
更新属性 el.setAttribute('x', 10) el.props.x = 10 → 重渲染时读到新值
响应式更新 VDOM diff → 最小 DOM 操作 全量重渲染 (全屏 1KB 数据，I2C 传输 ~25ms)
组件 function Comp() { return <div/> } function Comp() { return <text/> }
状态管理 useState / ref SolidJS createSignal

最大的区别：我们不 diff，而是 每次 signal 变化全量重绘。因为 128×64 = 1024 字节，通过 I2C（400kHz）传输只要 ~25ms，肉眼完全看不出闪烁。对于这类小屏幕，"全量重绘"是最简单可靠的方案。

九、如何扩展

加一个新组件，只需两步

1. 在 components.jsx 定义组件：

export function Line(props) {
  return <line x0={props.x0} y0={props.y0} x1={props.x1} y1={props.y1} />;
}

2. 在 oled-renderer.js 的 renderElement 里加渲染逻辑：

case 'line':
  oledDrawLine(el.props.x0, el.props.y0, el.props.x1, el.props.y1, 1);
  break;

当然，前提是 C 库里有对应的 oledDrawLine 函数，或者在 C 端加一个。

十、踩坑汇总

坑现象原因解决
ffi-napi 编译失败 Node 20 ARM64 上 C++ 编译报错 老 C++ 代码不兼容新版 node-api 换用 koffi（纯 JS FFI）
SolidJS 响应式不工作 setCount 了但 view 不更新 require('solid-js') 加载的是 server.cjs（SSR 版，createEffect 是空函数） 改用 solid-js/dist/solid.cjs
createEffect 初次不执行 OLED 初始化后不显示 createEffect 依赖 requestAnimationFrame，Node.js 没有 用 createRenderEffect（同步执行）
编译后 children 丢失 OLED 只显示 () => m() babel classic JSX 把 children 作为额外参数而不是 props.children jsx(type, props, ...children) 接受 rest 参数
信号泄漏 + 竞态 显示在 "0ount:" / "Count:0" 之间跳跃，最后乱码 组件在 createRenderEffect 里重复调用，每次创建新 signal 和 interval 组件只执行一次，返回 () => JSX 函数
babel-preset-solid universal 模式太复杂 需要实现 insert/setProp/effect/spread/memo 等全套接口 那是为复杂 DOM reconciler 设计的 改用简单的 @babel/plugin-transform-react-jsx + 自定义 pragma

十一、技术总结

这个项目本质上只做了三件事：

① JSX → 元素树： 自定义 jsx() 函数，把 babel 编译后的 JSX 调用拦截下来，构造自己的元素描述对象。
② 元素树 → OLED： renderElement() 递归遍历树，调用 C 驱动函数画到屏幕上。
③ 响应式刷新： createRenderEffect 追踪 signal，变了就全量重绘。

没有 VDOM diff、没有 fiber 调度、没有 DOM 操作 — 因为渲染目标是 1KB 的显存，最简单的方案就是最好的方案。
